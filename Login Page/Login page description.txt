OASIS ED登入界面

界面要素：
		1. 像素风格的OASIS ED
		2. 页面功能展示。背景视频循环播放，展示可交互片段。
		3. 滚动进入OASIS，添加过场动画

版面區塊：
- Hero/視覺：全屏背景影片 + 玻璃感遮罩，右側或中央放置登入卡片。
- 登入卡片：標題、副標、帳號/密碼欄位、記住我、忘記密碼、登入按鈕、替代登入（占位）。
- 狀態訊息區：右下或卡片底部顯示錯誤/成功/系統提示。
- 頁腳：版權、隱私/條款連結；預留語言切換。

互動/態樣：
- 按鈕：default/hover/active/disabled；hover 微放大 1.02，active 降低亮度 10%。
- 輸入框：focus 邊框高亮，錯誤時紅框與訊息；支援顯示/隱藏密碼。
- 背景：滑鼠移入顯示「可交互片段」按鈕，點擊可彈出簡述或局部放大。
- 滾動提示：向下箭頭，首屏滾動觸發過場。

狀態：
- 載入：登入按鈕顯示 loading spinner，表單 disabled。
- 錯誤：欄位紅框，文案「帳號或密碼錯誤」，連續 3 次可提示重設密碼。
- 成功：按鈕變色 + 勾選 icon，0.8s 後進入首頁/下一步並播放過場動畫。
- 空值：必填未填時按鈕 disabled，提示文案靠近欄位。

資產規格：
- 背景影片：mp4/webm，16:9，目標 720p；首屏載入 2-3 秒低碼率片段，完整版懶載。
- 後備圖：webp 1920x1080（<500KB）；低清 640px 預載模糊。
- Logo：SVG + 256px 方形 PNG。
- 圖標：24px PNG/SVG，像素風；統一描邊 1px。
- 字型：無版權像素風（例 Press Start 2P 替代），後備系統字型。
- 對比度：文字/背景達 4.5:1；按鈕文字 3:1。

可訪問性：
- 鍵盤：Tab/Shift+Tab 走訪欄位，Enter 提交，Esc 關閉彈出層。
- ARIA：輸入具 aria-label/aria-invalid，錯誤訊息 aria-live="polite"。
- 動畫減弱：偵測 prefers-reduced-motion，停用自動播放或改用靜態背景。
- 影片控制：提供暫停/播放按鈕且有可見焦點樣式。

UX 流程與轉場/狀態：
- 首屏載入：先顯示低清背景與骨架，影片低碼片段完成後淡入；登入卡片從 12px 上移 + 漸現 300ms。
- 背景互動：顯示「可交互片段」浮鈕，點擊彈出局部放大與 2-3 行說明；Esc/點遮罩可關閉。
- 滾動進場：向下滾動觸發過場，背景視差 + 光暈掃描，登入卡片固定位置；動畫時長 600-800ms。
- 表單提交：按鈕 loading 動畫 800-1200ms；成功則播放按鈕變色與勾選，延遲 0.8s 跳轉；失敗保留表單值並聚焦第一個錯誤欄位。
- 重試/鎖定：連續 3 次錯誤提示改用低強度文案，若需鎖定則顯示倒數提示與「重設密碼」鏈結。
- 無互動閒置：60 秒無互動淡出互動提示，保留輸入值；再互動時恢復。

Next.js 技術/儲存方案：
- 架構：Next.js App Router；頁面與背景影片使用 Server Component，表單與互動彈層採 Client Component。
- 資源載入：背景影片 preload="metadata" + poster 後備圖；完整版影片以懶載方式於視窗內觸發；互動段落的短片段（2-3s）作分段載入。
- 樣式：CSS Modules 或 Tailwind，保證像素風格（細格線、1px 邊框）；使用 next/font 匯入像素字型並備援系統字型。
- 圖片/影片儲存：影片與圖片放 CDN/遠端（webm/mp4 + webp），透過 cache-control 長效緩存；本地只放小型 placeholder。
- 資料：無後端時以假資料模擬登入（隨機延遲 400-900ms）；若需 API，使用 Route Handler 回傳靜態 JSON。
- 表單處理：前端同步驗證（必填、長度）；提交使用 fetch，錯誤/成功依前述狀態顯示。
- 性能：使用 next/script 對第三方腳本 lazyOnload；背景影片在 prefers-reduced-motion 時停用，自動切換後備圖。
- SEO/Meta：在 layout.tsx 設置基本 meta、OG 圖（採後備圖），設定 manifest/link favicon。
